This test plan was generated by exploring `http://eaapp.somee.com/Home/Dashboard`. The exploration identified key performance indicators (KPIs), data visualizations, and ranking components that require validation.

## 🏗️ Page Architecture
The Dashboard is a data-driven visualization page intended for administrators to monitor workforce metrics.

### 1. KPI Summary Cards
| Metric | Description | Expected Value Type |
| :--- | :--- | :--- |
| **Total Employees** | Count of all registered employees | Integer (e.g., 59) |
| **Average Salary** | Mean salary across all departments | Currency (e.g., $46,330) |
| **Average Age** | Mean age of the workforce | Float (e.g., 30.2) |
| **Monthly Salary Bill** | Total payroll expense for the current month | Currency (e.g., $2,733,480) |

### 2. Data Visualizations
- **Age Distribution**: Bar chart grouping employees by age brackets (20-29, 30-39, etc.).
- **Grade Breakdown**: Distribution by seniority levels (Junior, Middle, Senior, C-Level).

### 3. Ranking & Leaderboards
- **Top Earners**: Top 5 employees sorted by Salary (Descending).
- **Most Experienced**: Top 5 employees sorted by Tenure in months (Descending).

## 🧪 Test Cases

### Functional Tests
1. **TP-01: Data Synchronization**
   - **Goal**: Verify dashboard reflects the employee list.
   - **Steps**: Navigate to "Employees", add a new employee with a high salary, return to Dashboard.
   - **Expectation**: "Total Employees" increments, "Top Earners" updates if applicable.

2. **TP-02: Ranking Accuracy**
   - **Goal**: Validate sorting logic for "Top Earners".
   - **Steps**: Compare dashboard leaderboard with sorted data from the "Details" page.
   - **Expectation**: Ranking order matches actual salary values.

3. **TP-03: Calculation Logic**
   - **Goal**: Ensure average salary calculation is correct.
   - **Steps**: Fetch all employee salaries via API or List page, calculate average manually.
   - **Expectation**: Manual calculation matches Dashboard "Average Salary".

### UI & UX Tests
4. **TP-04: Chart Rendering**
   - **Goal**: Verify charts load and display tooltips.
   - **Steps**: Hover over "Age Distribution" bars.
   - **Expectation**: Data labels/tooltips appear with correct values.

5. **TP-05: Responsive Design**
   - **Goal**: Verify layout on mobile/tablet viewports.
   - **Steps**: Resize browser to 375x667 (iPhone SE).
   - **Expectation**: KPI cards stack vertically; charts remain legible.

### Security Tests
6. **TP-06: Authentication Guard**
   - **Goal**: Ensure dashboard is not accessible without login.
   - **Steps**: Clear cookies and navigate directly to `/Home/Dashboard`.
   - **Expectation**: Redirect to `/Account/Login`.
